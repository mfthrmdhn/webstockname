# Summary: Fix Login UI Centering

## What Changed

- `app/globals.css`: Changed from `@tailwind base; @tailwind components; @tailwind utilities;` to `@import "tailwindcss";`

## Why

Tailwind v4 (`@tailwindcss/postcss`) does not fully process v3-style `@tailwind` directives.
The compiled CSS was missing entire categories of utilities: colors (`bg-white`, `bg-gray-50`),
spacing (`p-8`, `mb-6`), sizing (`max-w-md`), and visual utilities (`rounded-lg`, `shadow-md`).
The login card had no visible styling — no background, no shadow, no constrained width.

The login page layout code was already correct (`min-h-screen flex items-center justify-center`).
The issue was purely that the necessary CSS classes were absent from the compiled bundle.

## Result

After rebuilding (`next dev` or `next build`), all Tailwind utilities will be generated correctly
and the login card will appear centered, styled, and visually correct.
