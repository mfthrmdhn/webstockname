## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding, item indicator positioning |
| sm | 8px | Compact element spacing (`p-2`, `gap-2`), badge padding |
| md | 16px | Default element spacing (`p-4`, `gap-4`), form field gaps |
| lg | 24px | Section padding (`p-6`), panel internal padding |
| xl | 32px | Layout gaps (`gap-8`), page-level padding (`p-8`) |
| 2xl | 48px | Major section breaks (used sparingly) |
| 3xl | 64px | Not used in Phase 2 UI surfaces |

**Exceptions:**
- POS panel height: `h-screen` (full viewport) — the two-panel layout occupies the full height
- Touch target minimum for cart +/- buttons: 40px (`h-10 w-10`) matching existing Button icon size
- Sidebar width: 256px (`w-64`) — carry forward from `AdminNav`

## Color

| Role | Tailwind Value | Hex Approx | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | `gray-50` / `white` | #F9FAFB / #FFFFFF | Page backgrounds, card surfaces, panel backgrounds |
| Secondary (30%) | `gray-100` / `gray-200` | #F3F4F6 / #E5E7EB | Table rows (hover), input backgrounds, sidebar secondary states, cart row hover |
| Accent (10%) | `blue-600` | #2563EB | Primary buttons, active nav links, focus rings, checkout button, Add Stock button |
| Destructive | `red-600` | #DC2626 | Remove item from cart (destructive icon button only) |

**Accent reserved for:**
1. "Complete Sale" / "Checkout" button (primary CTA in POS right panel)
2. "Add Stock" button in /admin/inventory table rows
3. Active navigation state (`bg-blue-50 text-blue-700 border-blue-200`)
4. Focus rings on all interactive elements (`focus:ring-2 focus:ring-blue-500`)

Accent is NOT used for: search results, cart items, payment method options, informational badges, salesperson picker items, or stock quantity display.

**Semantic additions for Phase 2:**
- `green-600` / `bg-green-50`: In-stock indicator (store_qty > 0) — used only in product search results and inventory table
- `amber-600` / `bg-amber-50`: Low stock indicator (store_qty > 0 but store_qty <= 5) — inventory table only
- `red-600` / `bg-red-50`: Out of stock indicator (store_qty === 0) — blocks adding to cart; also insufficient-stock error state
- `green-50 border-green-200`: Sale confirmation screen background — signals success

## Interaction Contracts

### Search debounce
- Debounce: 300ms after last keystroke before API call fires
- While fetching: show subtle loading indicator (spinner icon, 16px, `animate-spin`, `text-gray-400`) in search input right slot
- Minimum query length: 1 character (fire on any input)
- Source: Claude's Discretion from CONTEXT.md

### Cart quantity controls
- Display: `[–]` button | numeric display | `[+]` button
- `[–]` and `[+]` use `Button` variant="outline" size="icon" (`h-8 w-8`)
- Numeric display: `text-sm font-medium w-8 text-center` (not editable inline)
- Minimum quantity: 1 (pressing `[–]` at 1 is disabled, not remove)
- Remove item: separate `[x]` icon button (`Trash2` icon, `text-red-600`, `h-4 w-4`)
- Source: Claude's Discretion from CONTEXT.md

### Salesperson picker
- Rendered as `Select` (Radix UI) listing active CASHIER-role users
- Placeholder text: "Select salesperson..."
- Required: "Complete Sale" button remains `disabled` until a salesperson is selected
- Visual required indicator: asterisk (*) next to label, `text-red-600`
- Source: D-09, D-10 from CONTEXT.md

### Payment method selector
- Three options: Cash / Card / Transfer
- Rendered as a segmented button group (three adjacent `Button` components with `rounded-none` middle, full border radius on ends)
- Selected state: `bg-blue-600 text-white` (matching primary button)
- Unselected state: `border border-gray-300 bg-white text-gray-700 hover:bg-gray-50`
- Default: no pre-selection (cashier must choose)
- Cash selected: reveals "Amount Received" `Input` (type="number") below the selector
- Card / Transfer selected: "Amount Received" input is hidden (not rendered)
- Change due display: `text-lg font-semibold text-green-600` inline below the input
- Source: D-07, D-08 from CONTEXT.md

### Insufficient stock error
- Triggered: when POST /api/sales returns insufficient-stock error
- Display: `Alert` component variant="destructive" above the checkout button
- Copy: "Only [X] in stock — cannot complete this sale. Reduce the quantity for [Product Name] and try again."
- The alert is dismissible (X icon) and clears on next checkout attempt
- Source: D-15 from CONTEXT.md

### Sale confirmation screen
- Triggered: on successful POST /api/sales response
- Rendered as a `Dialog` (no close X — cashier must click the dismiss button)
- `DialogContent` background: `bg-green-50 border-green-200`
- Contents:
  - `DialogTitle`: "Sale Complete"
  - Sale total in display typography (`text-3xl font-bold`)
  - Itemized list: product name + quantity + line total
  - Salesperson credited: name
  - Payment method + (if Cash) amount received + change given
  - `Button` variant="default" full-width: "Start New Sale"
- "Start New Sale" click: closes dialog, clears cart, clears salesperson selection, clears payment selection, focuses search input
- Source: D-03 from CONTEXT.md

### Replenishment form validation
- Quantity field: required, integer, minimum 1, maximum = current warehouse_qty
- If quantity exceeds warehouse_qty: inline error "Only [X] units available in warehouse"
- Reason field: required, minimum 5 characters
- "Add Stock" button: disabled while fields invalid
- On success: Dialog closes, table row updates store_qty and warehouse_qty inline (optimistic update), toast: "Stock updated — [X] units moved to store floor"
- On error: toast with error message; dialog stays open
- Source: D-13 from CONTEXT.md

### StockBadge states
| Condition | Badge style | Text |
|-----------|-------------|------|
| store_qty > 5 | `bg-green-50 text-green-700 border-green-200` | "Store: [N]" |
| store_qty > 0 and <= 5 | `bg-amber-50 text-amber-700 border-amber-200` | "Store: [N]" |
| store_qty === 0 | `bg-red-50 text-red-700 border-red-200` | "Out of stock" |
| warehouse_qty > 0 | `bg-gray-100 text-gray-600` | "Warehouse: [N]" |

Badge: `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium border`

Products with `store_qty === 0` cannot be added to cart — clicking them shows no action (button disabled state, cursor-not-allowed).

## Accessibility Contract

- All interactive elements have visible focus rings (`focus-visible:ring-2 focus-visible:ring-blue-500`)
- Cart +/- buttons: `aria-label="Increase quantity"` and `aria-label="Decrease quantity"` respectively
- Cart remove button: `aria-label="Remove [product name] from cart"`
- Product search input: `aria-label="Search products"` with `role="combobox"` when result list is open
- Product result list: `role="listbox"`, each result `role="option"`
- Salesperson Select: Radix UI handles accessibility natively (keyboard navigation, ARIA attributes)
- Payment method buttons: group has `role="group" aria-label="Payment method"`, each button `aria-pressed` reflects selected state
- Sale confirmation Dialog: `aria-modal="true"`, focus trapped inside until dismissed (Radix UI default)
- StockBadge: `aria-label="Store stock: [N] units"` (screen reader cannot infer color meaning)
- Replenishment Dialog: focus moves to quantity input when dialog opens (`autoFocus` on input)
- Minimum touch target: 40px (`h-10 w-10`) for all icon-only buttons

## Implementation Notes for Executor

1. **POS layout must use `h-screen overflow-hidden` on the root wrapper** so both panels fill the viewport without a page scroll. Each panel has its own `overflow-y-auto`.

2. **Cart state lives in the POS page component** (`useState`). No external state manager. On confirmation dismiss, reset: `setCart([])`, `setSalesperson(null)`, `setPaymentMethod(null)`, `setAmountReceived('')`.

3. **Search uses `useEffect` with `setTimeout` for debounce** — 300ms. Clear the timeout on each keystroke. Fire `GET /api/products?q=[query]` only after the debounce resolves.

4. **Payment method segmented buttons** are three adjacent `<button>` elements styled to look joined. Use `rounded-l-md` on Cash, `rounded-none` on Card, `rounded-r-md` on Transfer, with `-ml-px` negative margin trick to collapse borders.

5. **Amount received field shows change due in real-time** as the cashier types — `changedue = parseFloat(amountReceived) - subtotal`. Display `Rp 0` if input is less than total (negative guard).

6. **Sale confirmation Dialog has no escape-key dismiss** — override Radix UI `onEscapeKeyDown` to `e.preventDefault()`. The cashier MUST click "Start New Sale" to prevent accidental dismissal mid-confirmation reading.

7. **Inventory table rows do optimistic updates** on replenishment success — update local state before the server response completes. Revert on error.

8. **AdminNav "Inventory" link** uses `Warehouse` icon from lucide-react and inserts between "Products" and "Audit Log" in the nav array.

9. **Currency formatting:** All monetary values rendered as `Rp [number]` with no decimal places for IDR. Use `new Intl.NumberFormat('id-ID').format(amount)` for thousands separators.
